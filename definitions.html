Roles
describe widgets that aren't otherwise available in HTML 5, such as sliders and tabs. The role of an element should not change. Instead, remove the original element and replace it with an element with the new role
Properties
describe characteristics of these widgets, such as if they are draggable, have a required element, or have a popup associated with them. `Property` values may change but are less likely to change than state values, this is not a rule
States
describe the current interaction state of an element, informing the assistive technology if it is busy, disabled, selected, or hidden. `State` values change frequently based on user interaction
Owned Elements
An 'owned element' is any DOM descendant of the element, any element specified as a child via aria-owns, or any DOM descendant of the owned child
Landmarks
Landmark roles (or "landmarks") programmatically identify sections of a page. Landmarks help assistive technology (AT) users orient themselves to a page and help them navigate easily to various sections of a page
